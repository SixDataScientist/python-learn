مشکل از نوع ورودی گرفتن بود، مثلا n,t رو با ایندکسشون جدا می‌کردم که بعد اومدم تغییر دادم
 و یه شکل دیگه ورودی رو گرفتم و n,t رو تو یه خط ورودی گرفتم برای i,j ها هم همینطور

شبیه تو نوشتم الان ینی با O(tn) همون 70 شد
دارم سعی میکنم از merge sort استفاده کنم بشه O(t log n)

تو یه آرایه ای ترکیبی از value ها و index ها داشته باشم و وقتی بر اساس value ها sort کردم
 index قبلیشون هم کنارشون داشته باشم
 که بتونم تشخیص بدم اگر تساوی داشتم index اش کوچکتر مساوی هست یا نه

 هم index و هم value رو نگه میدارم و
 براساس value مرتب میکنم در نهایت هم بر اساس جفت ]value ,index] باینری سرچ میزنم و تعداد رو بهم میده
